# Email Authentication Setup (one.com)

This guide configures SPF, DKIM, and DMARC for `arbeidmatch.no` when DNS is managed in one.com.

## 1) SPF record

### Target value

`v=spf1 include:send.one.com ~all`

### Steps in one.com

1. Log in to [one.com Control Panel](https://www.one.com/admin/).
2. Open **DNS** -> **DNS records**.
3. Click **Create record**.
4. Set:
   - **Type**: `TXT`
   - **Host/Name**: `@`
   - **Value/Text**: `v=spf1 include:send.one.com ~all`
   - **TTL**: default (or 3600)
5. Save.
6. If another SPF TXT exists on `@`, merge into one single SPF string (do not keep multiple SPF records).

---

## 2) DKIM setup

### Goal

Enable DKIM signing in one.com mail settings and publish the DKIM DNS record generated by one.com.

### Steps in one.com

1. Log in to one.com Control Panel.
2. Open **Email** -> **Mail settings** (or **Email settings** depending on UI).
3. Find **DKIM** / **Domain authentication**.
4. Click **Generate key** or **Enable DKIM**.
5. Copy the DNS details shown by one.com:
   - **Record type**: TXT (sometimes CNAME based on panel output)
   - **Host**: selector-based host (example format: `selector1._domainkey`)
   - **Value**: DKIM public key string (`v=DKIM1; k=rsa; p=...`)
6. Go to **DNS** -> **DNS records**.
7. Create the DKIM record exactly as provided by one.com.
8. Save and wait for DNS propagation.
9. Return to one.com DKIM page and click **Verify** / **Check status**.

---

## 3) DMARC record

### Target value

`v=DMARC1; p=quarantine; rua=mailto:post@arbeidmatch.no; pct=100`

### Steps in one.com

1. Open **DNS** -> **DNS records**.
2. Click **Create record**.
3. Set:
   - **Type**: `TXT`
   - **Host/Name**: `_dmarc`
   - **Value/Text**: `v=DMARC1; p=quarantine; rua=mailto:post@arbeidmatch.no; pct=100`
   - **TTL**: default (or 3600)
4. Save.

---

## 4) Validation checklist

After propagation (usually minutes to 24h):

1. Send a test email from `no-replay@arbeidmatch.no` to a Gmail/Outlook inbox.
2. Inspect message headers and confirm:
   - `SPF=pass`
   - `DKIM=pass`
   - `DMARC=pass` (or aligned pass)
3. Use external checkers:
   - MXToolbox SPF/DKIM/DMARC lookup
   - Google Admin Toolbox CheckMX
4. Confirm aggregate DMARC reports are received at `post@arbeidmatch.no`.

---

## Recommended rollout policy

- Start with current policy in this setup: `p=quarantine`.
- Once stable and false positives are low, consider `p=reject`.
- Keep `rua` mailbox monitored weekly.
